## 🚀 END-TO-END FLOW (PRODUCTION READY)

### 1. LANDING PAGE → "MULAI GRATIS"
- USER CLICK CTA
- REDIRECT KE `/register`
- OPTIONAL: TRACK REFERRAL / CAMPAIGN

---

### 2. REGISTER

INPUT:
- EMAIL
- PASSWORD

PROCESS:
- VALIDASI EMAIL FORMAT & PASSWORD POLICY
- HASH PASSWORD (BCRYPT / ARGON2)
- CREATE USER:
  - status = `UNVERIFIED`
- GENERATE EMAIL VERIFICATION TOKEN:
  - RANDOM, EXPIRY (15–60 MENIT)
- KIRIM EMAIL:
```

/verify-email?token=XYZ

```

OUTPUT:
- RESPONSE GENERIC:
"CHECK YOUR EMAIL TO ACTIVATE ACCOUNT"

---

### 3. EMAIL ACTIVATION

USER CLICK LINK:
```

/verify-email?token=XYZ

```

PROCESS:
- VALIDATE TOKEN:
  - EXISTS
  - NOT EXPIRED
  - NOT USED
- SET USER:
  - status = `VERIFIED`
- INVALIDATE TOKEN

OUTPUT:
- REDIRECT KE `/login` ATAU AUTO-LOGIN (OPSIONAL)

---

### 4. LOGIN

INPUT:
- EMAIL
- PASSWORD

PROCESS:
- VALIDATE CREDENTIAL
- CEK STATUS:
  - JIKA `UNVERIFIED` → BLOCK LOGIN
- GENERATE SESSION / JWT:
  - SHORT-LIVED ACCESS TOKEN
  - OPTIONAL: REFRESH TOKEN

SECURITY:
- RATE LIMITING
- GENERIC ERROR MESSAGE

OUTPUT:
- AUTHENTICATED SESSION

---

### 5. CREATE TENANT (SETELAH LOGIN)

INPUT:
- TENANT NAME

PROCESS:
- VALIDATE NAME (UNIQUE / SLUG)
- CREATE TENANT:
  - tenant_id
  - owner_id = user_id
- ASSIGN ROLE:
  - USER → OWNER
- SET DEFAULT SETTINGS

DATABASE RELATION:
```

users
tenants
user_tenants (user_id, tenant_id, role)

```

OUTPUT:
- REDIRECT KE DASHBOARD TENANT
